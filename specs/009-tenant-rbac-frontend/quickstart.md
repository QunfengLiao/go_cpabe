# 快速验证：租户 RBAC 前端授权与角色管理

本文用于计划完成后的实现验收。计划阶段不编写业务代码。

## 前置条件

- 后端已完成 008 租户级 RBAC，并执行包含 `010_tenant_rbac.sql` 的迁移和种子。
- 至少准备两个账号：账号 1 与账号 2 属于不同租户或拥有不同权限。
- 至少准备一个拥有 `tenant.role.read` 的用户、一个拥有 `tenant.role.manage` 的用户、一个只有只读权限的用户和一个普通成员。
- 桌面端依赖已安装。

## 验证场景

### 场景 1：角色管理菜单与路由

1. 使用拥有 `tenant.role.read` 的账号登录并选择租户。
2. 确认侧边栏显示“角色管理”菜单。
3. 使用没有 `tenant.role.read` 的账号登录同一租户或另一个租户。
4. 确认侧边栏不显示“角色管理”菜单。
5. 手动输入 `#/tenant/roles`。
6. 预期：无权限账号无法进入角色管理页面，显示 403 或跳转安全页。

### 场景 2：只读和管理权限差异

1. 使用只有 `tenant.role.read` 的用户进入角色管理页面。
2. 预期：可以查看角色列表和详情，不能创建、编辑、禁用或保存权限。
3. 使用拥有 `tenant.role.manage` 的用户进入页面。
4. 预期：可以创建自定义角色、编辑名称和描述、配置权限和禁用自定义角色。

### 场景 3：自定义角色和租户隔离

1. 在租户 A 创建 `SRE_ENGINEER` 自定义角色。
2. 确认该角色出现在业务角色分组。
3. 切换到租户 B。
4. 确认租户 B 的角色列表不显示租户 A 的自定义角色。
5. 在租户 B 创建同名 code。
6. 预期：如后端允许同 code 跨租户，则创建成功；同一租户重复 code 返回明确冲突。

### 场景 4：内置角色只读

1. 进入角色管理页面。
2. 查看 `TENANT_ADMIN`、`DO`、`DU` 等内置角色。
3. 预期：显示系统内置和只读状态。
4. 尝试编辑权限或禁用。
5. 预期：前端不提供可执行入口；若通过异常路径调用，后端错误被明确提示。

### 场景 5：权限选择器

1. 创建或编辑自定义角色权限。
2. 打开权限选择器。
3. 预期：只显示租户权限，不显示 `platform.*` 权限。
4. 使用搜索、分组全选和取消。
5. 保存后刷新角色详情，确认权限数量和权限列表更新。

### 场景 6：成员多角色弹窗

1. 使用拥有 `tenant.member.manage` 的用户进入成员页面。
2. 打开普通成员角色弹窗。
3. 同时选择 `DO` 和 `DU`，必要时再选择一个自定义业务角色。
4. 保存。
5. 预期：保存成功，成员列表刷新，两个能力角色同时保留。
6. 确认 `PLATFORM_ADMIN` 不出现在候选角色中。

### 场景 7：最后管理员保护

1. 找到当前租户最后一名 `TENANT_ADMIN`。
2. 尝试通过成员角色弹窗移除其管理员角色。
3. 预期：后端拒绝，前端显示“不能移除最后一名租户管理员”等明确错误。

### 场景 8：修改自身角色后的授权刷新

1. 使用有管理权限的用户修改自己的角色或其角色权限。
2. 保存成功后观察菜单和当前页面。
3. 预期：当前权限立即刷新；如果失去当前页面权限，系统跳转到安全页或显示 403。

### 场景 9：账号切换隔离

1. 登录账号 1，确认显示账号 1 的权限菜单。
2. 切换到账号 2。
3. 预期：切换过程中不短暂显示账号 1 的权限菜单。
4. 查看网络请求。
5. 预期：账号 2 的租户业务请求不携带账号 1 的 `tenant_id`。

### 场景 10：租户切换和刷新恢复

1. 在同一账号下切换租户。
2. 预期：权限和菜单立即按新租户重新计算。
3. 刷新 Electron 页面。
4. 预期：恢复正确账号、当前租户和授权上下文；恢复完成前不显示旧租户权限菜单。

### 场景 11：错误处理

1. 模拟后端返回 403。
2. 预期：前端显示权限不足提示。
3. 模拟网络失败。
4. 预期：授权状态进入错误，不保留为授权成功状态。
