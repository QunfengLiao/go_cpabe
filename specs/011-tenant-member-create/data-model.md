# 数据模型：租户管理员新增成员

## 复用实体

- `users`：全局登录账号；新建时写入用户名、邮箱、姓名、手机号、bcrypt 密码摘要、有效状态和首次改密标记。
- `tenant_users`：账号在当前租户的有效成员关系；重复操作恢复或保持 `active`。
- `user_roles`：当前租户的 `DO`、`DU` 授权；不得通过本功能写入管理员或平台角色。
- `audit_logs`：记录操作者、租户、目标用户、创建或复用结果，不保存密码和请求体。

## 状态规则

- 新邮箱：创建 `users` → 建立 `tenant_users` → 授予角色 → 审计。
- 已有邮箱：复用 `users` → 建立或恢复 `tenant_users` → 补齐角色 → 审计。
- 已禁用用户：拒绝加入，不能借成员创建重新启用全局账号。
