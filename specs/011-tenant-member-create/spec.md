# 功能规格：租户管理员新增成员

**功能目录**：`011-tenant-member-create`  
**创建日期**：2026-07-12  
**状态**：已确认

## 用户场景与验收

### 用户故事 1：创建租户成员账号（P1）

作为租户管理员，我希望在当前租户的成员页面创建数据拥有者或数据使用者账号，使新成员能够登录并参与文件加密共享。

**独立验收**：租户管理员填写账号资料并选择 `DO`、`DU` 中至少一个角色后，成员出现在当前租户列表；使用初始密码可以登录，且账号被要求修改密码。

1. **假设** 邮箱尚未注册，**当** 租户管理员不填写密码并创建成员，**那么** 系统使用初始密码 `lqf999..`，只保存不可逆密码摘要，并设置首次改密标记。
2. **假设** 邮箱已有账号，**当** 租户管理员新增成员，**那么** 系统复用账号、加入当前租户并授予所选角色，不修改原密码和资料。
3. **假设** 操作者不具备成员管理权限，**当** 调用新增成员能力，**那么** 系统拒绝且不创建账号、成员关系或角色。

### 用户故事 2：成员立即进入加密接收者链路（P2）

作为数据拥有者，我希望新建的 DU 完成公钥登记后能出现在 RSA 接收者列表中，从而选择其作为文件接收者。

**独立验收**：新建 DU 登录并登记 RSA 公钥后，拥有文件上传权限的 DO 刷新接收者列表即可选择该 DU。

## 边界情况

- 用户名或邮箱重复但指向不同账号时拒绝创建，不能覆盖已有身份。
- 已存在邮箱只复用账号；请求中的姓名、手机号和密码均不能覆盖原资料。
- 角色只能选择当前租户允许的 `DO`、`DU`，不得通过该入口授予租户管理员或平台角色。
- 至少选择一个业务角色；重复提交应得到同一成员事实，不产生重复关系或重复角色。
- 禁用租户、禁用账号或跨租户操作必须拒绝。

## 功能需求

- **FR-001**：租户管理员成员页必须提供“新增成员”入口和账号资料表单。
- **FR-002**：新增成员必须接受用户名、姓名、邮箱、可选手机号以及 `DO`、`DU` 角色集合。
- **FR-003**：新账号必须使用固定初始密码 `lqf999..`，保存时不得保留明文，并必须标记首次登录修改密码。
- **FR-004**：邮箱已有账号时必须复用账号且不得重置密码或覆盖资料。
- **FR-005**：必须在可信当前租户内建立有效成员关系并授予所选业务角色。
- **FR-006**：接口必须要求 `tenant.member.manage` 权限，不信任请求中的租户标识。
- **FR-007**：创建或复用成员必须写入脱敏审计，不记录密码、密码摘要或完整请求体。
- **FR-008**：响应只返回成员展示信息、是否新建账号和仅在新建时显示的初始密码提示。
- **FR-009**：新建 DU 在自行登记有效 RSA 公钥后必须进入当前租户接收者目录。

## 成功标准

- **SC-001**：租户管理员能在 2 分钟内完成成员创建和角色选择。
- **SC-002**：新账号使用初始密码成功登录率为 100%，并显示首次改密状态。
- **SC-003**：已有账号加入租户后原密码保持有效，密码被意外重置的测试数为 0。
- **SC-004**：越权、跨租户和非法角色请求在任何成员或角色写入前全部被拒绝。

## 假设与范围

- 本期复用现有用户、租户成员和角色模型，不新增数据库表。
- 固定初始密码仅用于工程演示；生产环境必须改为随机一次性凭据或邀请链接。
- 本期不实现邮件邀请、短信通知和管理员代重置密码。
- 本功能不触及 Crypto、Policy 和 Benchmark 算法实现；只通过既有 RSA 接收者目录验证集成。

## 宪章符合性

本功能不修改混合加密和真实 CP-ABE 边界；成员身份仍由 User 模块管理，加密接收者仍由 Crypto/RSA 公钥目录读取。安全相关代码必须说明固定初始密码、账号复用、租户隔离和审计边界。
