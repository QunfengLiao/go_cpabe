# 桌面端契约：后端调用封装

## 目标

桌面端必须保留统一的后端调用入口，避免页面组件直接散落请求地址和请求细节。基础工程阶段至少预留健康检查调用能力。

## 推荐模块

```text
desktop/src/renderer/api/
  config.ts
  client.ts
  health.ts
```

## 行为约定

- `config.ts` 集中管理 `DESKTOP_API_BASE_URL` 或默认后端地址。
- `client.ts` 统一处理请求发起、响应解析和错误归一化。
- `health.ts` 暴露健康检查调用，供首页后续接入。

## 错误处理

- 网络不可达时，调用层返回安全的错误对象或错误消息。
- 页面展示错误时不得显示敏感配置。
- 后端地址不得硬编码在多个页面组件中。

## 范围边界

- 本阶段不封装用户、文件、策略、加密、审计等业务接口。
- 本阶段不引入复杂状态管理或 UI 框架。
