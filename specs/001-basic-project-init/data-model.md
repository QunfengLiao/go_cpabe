# 第 1 阶段数据模型：基础工程初始化

本阶段不创建业务数据表，不设计用户、文件、策略、审计或基准测试实体。以下模型用于配置、健康检查、桌面端初始化和文档契约。

## 本地环境配置

**含义**：开发者本机用于启动后端和桌面端的配置集合。真实敏感值只存在于本地 `.env` 或系统环境变量中，不进入仓库。

**字段**：

- `APP_ENV`：运行环境标识，示例值为 `development`。
- `APP_PORT`：后端监听端口，示例值为 `8080`。
- `MYSQL_HOST`：远程 MySQL 主机，占位值不得是真实服务器地址。
- `MYSQL_PORT`：远程 MySQL 端口，示例值为 `3306`。
- `MYSQL_USER`：MySQL 用户名，占位值不得是真实账号。
- `MYSQL_PASSWORD`：MySQL 密码，占位值不得是真实密码。
- `MYSQL_DATABASE`：MySQL 数据库名，占位值可使用说明性示例。
- `REDIS_ADDR`：Redis 地址，格式为 `host:port`，占位值不得是真实服务器地址。
- `REDIS_PASSWORD`：Redis 密码，占位值不得是真实密码；无密码时可为空。
- `REDIS_DB`：Redis 逻辑库编号，示例值为 `0`。
- `DESKTOP_API_BASE_URL`：桌面端访问后端的基础地址，示例值为 `http://localhost:8080`。

**校验规则**：

- 必填配置缺失时，后端启动或健康检查必须给出可排查错误。
- 端口和 Redis 逻辑库必须能解析为数字。
- 日志、响应和文档示例不得输出真实密码、令牌或完整敏感连接串。

## 健康检查结果

**含义**：开发者确认后端和远程依赖是否可用的状态集合。

**字段**：

- `status`：整体状态，取值建议为 `ok` 或 `degraded`。
- `checkedAt`：检查时间。
- `app.status`：服务自身状态。
- `app.env`：当前运行环境。
- `mysql.status`：MySQL 状态，取值建议为 `ok` 或 `error`。
- `mysql.message`：安全摘要，不包含真实密码或完整连接串。
- `redis.status`：Redis 状态，取值建议为 `ok` 或 `error`。
- `redis.message`：安全摘要，不包含真实密码或完整连接串。

**状态规则**：

- app 正常、MySQL 正常、Redis 正常时，整体状态为 `ok`。
- 任一依赖不可用时，整体状态为 `degraded`，但服务仍应返回结构化健康检查结果。
- 依赖错误只返回分类和简要原因，不返回原始敏感配置。

## 桌面端基础首页

**含义**：桌面端启动后的默认界面，用于确认桌面端工程运行正常。

**字段/展示内容**：

- 项目名称：CP-ABE 加密文件共享系统。
- 后端连接状态占位：显示未检查、检查中、可用或不可用等状态。
- 基础说明：仅用于开发环境确认，不进入业务流程。

**规则**：

- 首页不得依赖后端成功连接才能渲染。
- 首页不得引入复杂业务功能或权限逻辑。

## 后端调用封装

**含义**：桌面端访问后端的统一入口，后续业务功能复用该入口。

**字段/职责**：

- `baseUrl`：集中管理后端基础地址。
- `request`：统一发起 HTTP 请求。
- `getHealth`：预留健康检查调用。
- `ApiError`：统一表达请求失败原因。

**规则**：

- API 地址不得散落在多个页面组件中。
- 请求失败必须能被页面捕获并显示为安全、简短的状态。

## 项目启动说明

**含义**：根目录 README 中面向新开发者的运行指南。

**内容要求**：

- 环境变量配置步骤。
- 后端启动和测试命令。
- 桌面端启动和验证命令。
- 健康检查访问方式。
- 常见问题排查：配置缺失、网络不可达、认证失败、依赖未安装。
- 安全提醒：不得提交真实 `.env` 和个人服务器凭据。
