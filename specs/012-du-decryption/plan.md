# 实施计划：DU 密钥与本地解密闭环

## 技术方案

- 后端提供当前可见文件的密钥信封集合、密文流和兼容收到文件列表；接口只约束租户、文件可见性和对象状态。
- Electron 主进程负责密文临时文件、系统安全存储私钥读取、按 `key_id`/公钥指纹匹配信封、保存对话框和 Worker 生命周期。
- Go Worker 负责 RSA-OAEP-SHA256 解封装与 GCPABE01 分块 AES-GCM 解密；找不到私钥或失败时保留原始密文包并删除部分明文。
- React 新增“我的密钥”和“收到的文件”，菜单只按 `crypto.key.self.manage` 与 `file.read` 控制可见性，不使用 `file.decrypt.invoke` 作为解密门槛。

## 宪章与注释

继续使用混合加密；RSA 只解封装 DEK。所有新增 Go 函数/方法使用中文前置注释和 GoDoc 前缀，重点解释租户/文件可见性边界、密钥信封不是 RBAC 解密授权、私钥边界、完整性失败和原始密文包保留。服务端不实现密码算法。
