# 前端 UI 设计规范

本文档是 React + Electron 前端页面开发和重构的唯一视觉规范。新增页面应优先复用公共组件和设计 Token；确有业务差异时，通过组件参数表达差异，不在页面中复制一套视觉规则。

## 1. 产品定位

本产品是企业级数据安全和密码学文件管理客户端。视觉应专业、克制、清晰、稳重，保持适中的信息密度，接近企业安全控制台，不采用营销官网风格，不使用花哨渐变和大面积装饰。

## 2. 页面结构

页面统一采用以下结构：

```text
PageShell
├─ PageHeader
│  ├─ 标题
│  ├─ 一句话说明
│  └─ 页面级操作
├─ PageTabs（可选）
└─ ContentCard
   ├─ FilterToolbar（可选）
   ├─ Content
   └─ Pagination（可选）
```

页面最大宽度、标题间距、内容卡片圆角和区块间距由公共 Token 统一控制，禁止每个页面重新定义一套标题和内容宽度。

## 3. 设计 Token

颜色、字体层级、间距、圆角、阴影、控件高度、边框颜色、状态色和页面最大宽度必须通过 `styles.css` 中的 CSS 变量使用。页面和组件不得散落颜色值、阴影值和布局魔法数字。

固定数值只允许用于图标、边框、圆角、控件高度、头像、间距 Token 和最小可用宽度。组件尺寸稳定，页面结构弹性。

## 4. 弹性布局

页面布局优先使用 CSS Grid、Flex、`fr`、`minmax()`、`clamp()`、`max-content`、`fit-content`、容器查询、`min-width: 0` 和文本省略。固定宽度不得锁死整张表格，不得使用 `word-break: break-all` 处理普通文本或指纹，不得通过压缩列宽造成逐字换行、按钮裁切或横向溢出。

内容空间不足时，优先按内容容器宽度使用 `@container` 降级，而不是只依赖浏览器窗口媒体查询。

## 5. 响应式降级顺序

1. 收缩弹性列。
2. 长文本单行省略，并通过 Tooltip 查看完整内容。
3. 简化头像和辅助文字。
4. 将低频操作放入更多菜单。
5. 隐藏低优先级列，并将其放入详情抽屉。
6. 必要时转为紧凑卡片结构。

禁止将表头或数据压成竖排。

## 6. 页面操作层级

每个页面最多一个主要操作，每行最多一个主要操作。其他操作使用次级图标按钮、更多菜单或危险操作样式。下载、详情、保存密文、打开文件夹等不能全部以同等级按钮长期并排展示。

## 7. 表格

表头弱化，行高统一，主要信息获得剩余空间；时间、状态和操作保持单行；长文本使用省略和 Tooltip；缺失值使用“未知”或“暂无”；操作列不得换行。筛选工具栏和表格属于同一视觉容器，Hover 只使用轻微背景变化，避免大量彩色 Badge。

## 8. 状态和文案

状态标签只用于成功、处理中、警告、失败、已停用和当前使用等真实状态。普通属性使用弱化文本，不全部制作成 Badge。页面文案使用中文，不向用户展示 `ACTIVE`、`UNAUTHORIZED`、`can_decrypt` 或 `authorization_status`。

## 9. 完整页面状态

数据页面必须覆盖 Skeleton Loading、Empty、Search Empty、Error、Normal、Disabled 和 Processing 状态。状态处理应由页面状态和公共状态组件共同完成，不能只实现正常数据状态。

## 10. 公共组件约束

公共组件位于 `desktop/src/renderer/src/components/ui/`，优先复用：`PageShell`、`PageHeader`、`PageTabs`、`ContentCard`、`FilterToolbar`、`DataTable`、`FileNameCell`、`FingerprintCell`、`AlgorithmDisplay`、`OwnerCell`、`StatusBadge`、`RowActions`、`DetailDrawer`、`EmptyState`、`ErrorState`、`LoadingSkeleton` 和 `ConfirmDialog`。

页面业务组件只负责数据获取、事件处理和业务字段映射；公共组件负责结构、间距、状态表现和响应式行为。不要为“全部密文”和“我的加密文件”复制两套列表。

## 11. 验收尺寸

重构后的页面至少检查 1366×768、1440×900、1920×1080，分别验证侧边栏展开和折叠，以及 Windows 125% 和 150% 缩放。必须确保表头、指纹、密文大小、时间和操作按钮不逐字换行、不被裁切；筛选栏在桌面宽度下不异常拆行；页面不会同时出现大片空白和局部拥挤。
